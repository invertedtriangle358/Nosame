# 野雨 -Nosame-

野雨（のさめ）は、ブラウザで動作する Nostr SNS の縦書き短文クライアントです。

秘密鍵をアプリに入力する必要はありません。投稿、リアクション、リポストなどの署名は、NIP-07 対応のブラウザ拡張機能を通して行います。

---

## 特徴

### 縦書きインターフェース

- タイムライン、投稿フォームを縦書きで表示します。
- フォントには主に「さわらび明朝」を使用しています。
- タイムラインは横方向にスクロールします。

### 短文投稿

- 投稿本文の表示文字数は最大 108 文字です。入力中に文字数をリアルタイム表示します。
- URL やハッシュタグを投稿タグとして付与します。
- 引用参照を含む投稿にも対応しています。

### Nostr 基本機能

- kind 1 のテキスト投稿を表示します。
- kind 6 のリポストを表示・送信できます。
- kind 7 のリアクションを送信できます。
- プロフィール metadata を取得し、表示名やアイコンを反映します。
- プロフィール画面では対象ユーザーの投稿を個別に取得できます。

### フィルタリング

- NG ワードを含む投稿を非表示にできます。また、 NG ワードを含む内容の投稿も防ぎます。
- hex 公開鍵または npub を指定して、任意の公開鍵を遮断できます。
- NIP-36 `content-warning`、`nsfw` タグ、本文中の `#nsfw` を非表示にできます。

### リレー管理

- 設定パネルから接続リレーを追加・削除・編集できます。
- リレー編集は保存ボタンでまとめて反映します。
- 一時購読は `EOSE` またはタイムアウトで閉じます。
- 追加取得の件数や同時一時購読数に上限を設けています。

### 安全性と負荷対策

- NIP-07 を使用し、秘密鍵をアプリ側で扱いません。
- 受信イベントの形式、ID、署名を検証します。
- 署名検証前に未知の購読 ID を破棄します。
- 巨大な WebSocket メッセージや巨大な event content を破棄します。
- プロフィール取得では要求した pubkey の metadata だけを受理します。
- 参照イベント取得では要求した event id だけを受理します。
- 投稿送信後はリレーの `OK` 応答を確認します。
- イベント、参照イベント、プロフィールキャッシュには保持上限を設けています。

---

## 初期接続リレー

デフォルトでは以下のリレーに接続します。

- `wss://relay-jp.nostr.wirednet.jp`
- `wss://yabu.me`
- `wss://r.kojira.io`
- `wss://nostr.compile-error.net`

接続するリレーは、設定から変更できます。

---

## 動作要件

- 推奨ブラウザ: Firefox
- 必須: NIP-07 対応の Nostr ブラウザ拡張機能

例:

- nos2x
- Alby
- nos2x-fox

## ファイル構成

プレーンな JavaScript ES Modules と CSS で構成されています。

```text
.
├── index.html          # HTML構造
├── style.css           # 縦書きレイアウト、テーマ、UIスタイル
├── main.js             # エントリーポイント、各クラスの初期化
├── config.js           # 定数、UI文言、初期設定
├── nostr-core.js       # Nostr関連クラスの再エクスポート
├── nostr-client.js     # リレー接続、購読、送信、OK応答処理
├── nostr-codec.js      # npub / nevent などのエンコード・デコード
├── nostr-stores.js     # localStorage管理、プロフィールキャッシュ
├── nostr-validator.js  # イベント検証、署名検証、フィルタ判定
└── ui.js               # DOM操作、タイムライン描画、設定画面
```

---

## 主な設定値

主な制限値は `config.js` で管理しています。

- `MAX_POST_LENGTH`: 表示文字数の上限
- `NOSTR_REQ_LIMIT`: 通常タイムライン取得件数
- `NOSTR_REQ_SINCE_SECONDS_AGO`: 通常タイムラインの取得対象期間
- `MAX_RELAY_MESSAGE_BYTES`: 受信 WebSocket メッセージサイズ上限
- `MAX_EVENT_CONTENT_BYTES`: 通常イベント content のサイズ上限
- `MAX_METADATA_CONTENT_BYTES`: metadata content のサイズ上限
- `MAX_ONE_SHOT_SUBSCRIPTIONS`: 同時一時購読数の上限
- `EVENT_ACK_TIMEOUT_MS`: 投稿後の `OK` 応答待ち時間

---

## 注意

野雨は開発中のクライアントです。Nostr の仕様やリレーの挙動によって、表示・投稿・取得結果が変わる場合があります。

公開リレーへ過度な負荷をかけないよう、取得件数、一時購読、再接続、保持件数には上限を設けています。
