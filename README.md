# 野雨 -Nosame-

「野雨（のさめ）」は、ブラウザで動作するNostrプロトコル対応の縦書き短文クライアントです。  
Nostr 秘密鍵の入力は不要で、NIP-07拡張機能による安全な投稿・閲覧が可能です。

---

## 主な機能と特徴

### 1. 縦書きインターフェース
- タイムラインおよび投稿フォームはすべて縦書きで表現。
- フォントには主に「さわらび明朝」を採用。
- タイムラインは横方向にスクロール。

### 2. 制限と仕様
- **表示文字数制限** : 最大108文字。入力時にはリアルタイムに文字数がカウントされます。
- **取得制限** : 直近1時間（3600秒）前からの投稿、または最大 100 件までのノートを素早く読み込みます。

### 3. 安心・安全なセキュリティ
- アプリケーションに直接秘密鍵を入力する必要はありません。投稿時やリアクション時には、ブラウザのNostr拡張機能（NIP-07 拡張機能）を介して安全にイベント署名を行います。

### 4. コンテンツ管理・フィルタリング機能
- **ユーザーNGワード(禁句)** : 登録した禁句を含む投稿をタイムラインから除外します。また、禁句を含む内容の投稿を未然に防ぎます。
- **公開鍵のブロック(遮断)** : 特定の hex 公開鍵または npub を指定して、そのユーザーの投稿を非表示にできます。
- **NIP-36 Content Warning / #NSFW 非表示対応** : 設定から「閲覧注意を非表示」を有効にすることで、NIP-36 `content-warning` 設定された投稿や、本文に `#nsfw` が含まれる投稿をタイムラインから除外可能です。

---

## 初期接続リレー一覧

デフォルトでは、以下の日本国内の主要なNostrリレーに自動で接続します。
- `wss://relay-jp.nostr.wirednet.jp`
- `wss://yabu.me`
- `wss://r.kojira.io`
- `wss://nostr.compile-error.net`

※ 接続するリレーは、アプリケーション内の設定パネルからいつでも追加・削除が可能です。

---

## 動作要件

- **推奨ブラウザ** : Firefox
- **必須要件** : NIP-07 に対応した Nostr 拡張機能（例: nos2x, Alby など）が導入されていること。

---

## ファイルの構成

外部ライブラリへの依存を最小限に抑え、プレーンな JavaScript (ES Modules) と CSS で構築されています。

```text
.
├── index.html         # HTML構造・構造定義
├── style.css          # レイアウト、テーマ、デザイン
├── main.js            # エントリーポイント・各クラスの初期化
├── config.js          # 定数、テキストメッセージ、初期設定の管理
├── nostr-core.js      # Bech32(npub)変換、イベント検証ロジック
├── nostr-client.js    # Nostr通信
├── nostr-codec.js     #
├── nostr-stores.js    #
├── nostr-validator.js #
└── ui.js              # DOM操作、タイムライン描画、設定画面ハンドリング

```
